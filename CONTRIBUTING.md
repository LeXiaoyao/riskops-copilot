# 贡献指南 / Contributing

本项目为**个人作品集与学习项目**，当前阶段不接受外部代码贡献（Pull Request）。但欢迎以下形式的参与：

> This project is a **personal portfolio / learning project**. External code contributions (PRs) are not accepted at this stage. However, the following are welcome:

---

## 反馈渠道 / Feedback Channels

### 1. Issues

通过 [GitHub Issues](https://github.com/LeXiaoyao/riskops-copilot/issues) 反馈：
- 业务建议（指标口径、归因框架、合规规则）
- 技术建议（架构、技术栈、性能）
- 文档问题（错别字、链接失效、表述不清）
- 想法讨论（产品方向、Demo 故事线）

请使用对应的 Issue 模板（见 `.github/ISSUE_TEMPLATE/`）。

### 2. 邮件

复杂的业务讨论、合作意向可发邮件至：xiaoyao201207@gmail.com

---

## Issue 提交规范 / Issue Guidelines

- 提交前先**搜索已有 Issue**，避免重复。
- 标题简洁明了，能反映核心问题。
- 描述包含：背景 / 现状 / 期望 / 复现步骤（如适用）。
- 涉及业务的 Issue，请说明业务场景和真实痛点。

---

## 文档贡献 / Documentation Contributions

如果发现文档错误或想补充内容，可以：

1. **小问题**（错别字、链接）：直接开 Issue 描述。
2. **结构性建议**（章节调整、表达优化）：开 Issue 讨论后再决定是否落地。
3. **PRD 章节补强**：参考 PRD v6 §14.4 的 T-01 到 T-10 待办清单。

---

## 行为准则 / Code of Conduct

- 保持专业和尊重。
- 对事不对人。
- 不发广告、不发与项目无关的内容。
- 不强求作者必须采纳建议——这是个人项目，作者保留最终决定权。

## 关于 Fork

欢迎 Fork 本项目用于：
- 学习参考
- 改造为自己的项目
- 商业用途（受 [MIT License](LICENSE) 约束）

如果你的衍生项目从本项目获益较大，**期望但不强制**：
